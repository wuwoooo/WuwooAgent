# 微信聊天分析接口文档（FastAPI）

---

## 一、服务概述

该服务用于：

> **（推荐）客户端本地 OCR → 上传识别文本 → 调用 AI Agent 生成回复建议**；或兼容旧版 **上传聊天截图**（服务端不对图片做 OCR 时走占位描述）。

当前为 **MVP**：支持 **客户端本地 OCR 后上传 `ocr_text`**（推荐）或 **仅上传截图**（兼容）；服务端 **不对图片做 OCR**，已 **调用腾讯云 ADP 智能体** 生成 `reply_text`。

## 三、接口地址

**路径关系（与当前 Nginx 一致）：**

| 场景 | 浏览器 / 客户端应请求的 path | 实际到达 uvicorn 的 path |
| ---- | ---------------------------- | ------------------------- |
| **走 Nginx（正式、推荐）** | **`/api/wechat/chat`**（必须带 `/api`） | **`/wechat/chat`** |
| **直连 8000（仅调试）** | **`/wechat/chat`**（无 `/api`，因不经过 Nginx） | **`/wechat/chat`** |

说明：Nginx 配置为 `location /api/` + `proxy_pass http://127.0.0.1:8000/;` 时，会把请求 URI 里的 **`/api/` 段去掉**再转发，因此 **`/api/wechat/chat` → `/wechat/chat`**，与 FastAPI 中 `@app.post("/wechat/chat")` 一致。

### 推荐（经 Nginx，对外请始终用带 `/api` 的地址）

```text
http://118.24.71.189/api/wechat/chat
```

### 直连 uvicorn（绕过 Nginx 时，不要写 `/api`）

```text
http://118.24.71.189:8000/wechat/chat
```

## 四、接口说明

### 1️⃣ 接口名称

```text
对外（经 Nginx）：POST /api/wechat/chat
应用内（uvicorn）：POST /wechat/chat
```

---

### 2️⃣ 请求类型

```text
multipart/form-data
```

---

### 3️⃣ 请求参数

**仍为 `multipart/form-data`，但支持两种用法（二选一）：**

| 参数名        | 类型   | 必填 | 说明 |
| ------------- | ------ | ---- | ---- |
| session_id    | String | ✅   | 会话 ID（用于上下文） |
| contact_name  | String | ✅   | 联系人名称 |
| ocr_text      | String | 条件 | **客户端本地 OCR 后的全文**；与 `image` 二选一，至少提供其一 |
| image         | File   | 条件 | 聊天截图；与 `ocr_text` 二选一，至少提供其一 |

说明：

- **推荐（Agent IME 本地 OCR）**：只传 `ocr_text` + `session_id` + `contact_name`，**不传 `image`**。
- **兼容旧版**：只传 `image` + `session_id` + `contact_name`（服务端仍保存图片；智能体侧暂用占位描述，未对图片做 OCR）。
- **同时传 `ocr_text` 与 `image`**：以 `ocr_text` 为准（仍会保存图片文件）。
- **`image` 字段若存在但内容为空**：视为未上传图片（与未传 `image` 一致）。

---

### 4️⃣ 请求示例（前端）

**方式 A：仅传本地 OCR 文本（推荐）**

```javascript
const formData = new FormData();
formData.append("ocr_text", "（从截图识别出的聊天文字……）");
formData.append("session_id", "123");
formData.append("contact_name", "张三");

fetch("/api/wechat/chat", {
  method: "POST",
  body: formData,
})
  .then((res) => res.json())
  .then((data) => console.log(data));
```

**方式 B：上传截图（兼容旧版）**

```javascript
const formData = new FormData();
formData.append("image", file);
formData.append("session_id", "123");
formData.append("contact_name", "张三");

fetch("/api/wechat/chat", {
  method: "POST",
  body: formData,
})
  .then((res) => res.json())
  .then((data) => console.log(data));
```

## 五、返回结果

### 示例返回

```json
{
  "ok": true,
  "messages": [
    {
      "role": "user",
      "text": "五一想去云南玩"
    },
    {
      "role": "assistant",
      "text": "可以呀，你们几个人出行？"
    }
  ],
  "reply_text": "可以呀，你们大概几个人出行？我先按轻松一点的玩法帮你看看。"
}
```

---

### 字段说明

| 字段       | 类型    | 说明       |
| ---------- | ------- | ---------- |
| ok         | Boolean | 是否成功   |
| messages   | Array   | 聊天上下文 |
| reply_text | String  | AI生成回复 |

**说明：** 成功时 `messages[0].text` 为**发给智能体的完整用户侧文案**（含联系人前缀与 OCR 包装说明），不一定等于客户端原始 `ocr_text` 纯文本；`messages[1].text` 为智能体回复。

### 失败时（HTTP 4xx / 5xx）

| HTTP | 条件 | 响应体补充 |
| ---- | ---- | ---------- |
| 400  | 既无有效 `ocr_text` 也无非空 `image` | `ok: false`，`error` 说明原因，`reply_text` 为空字符串 |
| 503  | ADP 调用失败（密钥、网络、智能体报错等） | `ok: false`，`error` 为错误信息，`reply_text` 为空字符串 |

---

http://118.24.71.189:8000/health

## 六、后端处理流程

当前逻辑：

```text
1. 校验：ocr_text 与 image 至少提供其一，否则 400
2. 若含 image：保存到服务器（uploads 目录）
3. 构造发给智能体的 user 文本：优先使用 ocr_text；否则使用「仅有截图」占位描述
4. 调用 ADP 智能体生成 reply_text 并返回 JSON
```

## 十、Nginx 配置（关键）

与 FastAPI 路由 **`/wechat/chat`**、**`/health`** 配合时，使用**带末尾斜杠**的 `proxy_pass`，会去掉 `/api` 前缀再转发：

```nginx
location /api/ {
    proxy_pass http://127.0.0.1:8000/;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
}
```
